# Scoring

## Important

- You can only obtain either On-time submit or Late submit, not both.
- You can only submit once, multiple submits does not increase your score.

## Scoring rules

| Score Rule | Points |
| --- | --- |
| On-time submit | +300 points |
| Late submit | +150 points |
| Each bonus algorithm | +50 points |
| Well explained code. <br/>If the code is well explained, with algorithms complexity and how it works | up to +100 points<br>(minimum of +50 points) |
| Code does not compile | -50 points |
| Program does not execute | -50 points |

## Penalties

Other penalties might be applied based on the mods judgement.

Any other algorithms not listed here will not be considered, no matter how impressive it might be.

## Important

If none of the three mandatory algorithms are implemented correctly, the score will be:

**0 points**

Regardless of completed bonuses or any other quality.

# Evaluation Criteria

Solutions will be evaluated considering:

- Correctness of the algorithm
- Code readability
- Project organization
- Clarity of documentation
- Appropriate use of C++
- Explanation of how the algorithm works

The shortest solution is not necessarily the best.
The most complex solution is also not necessarily the best.
The main objective is to demonstrate understanding of the problem and the implemented algorithm.

Good luck and have **fun** learning.
